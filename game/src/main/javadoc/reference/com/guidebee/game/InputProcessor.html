InputProcessor

com.guidebee.game.InputProcessor
Known Indirect Subclasses
GestureDetector, InputAdapter, InputMultiplexer, InputProcessorQueue, LayerManager, RemoteSender, Stage, UIWindow
GestureDetector InputProcessor implementation that detects gestures (tap, long press, fling, pan, zoom, pinch) and hands them to a GestureDetector.GestureListener. 
InputAdapter An adapter class for InputProcessor. 
InputMultiplexer An InputProcessor that delegates to an ordered list of other InputProcessors. 
InputProcessorQueue Queues events that are later passed to the wrapped InputProcessor. 
LayerManager The LayerManager manages a series of Layers. 
RemoteSender Sends all inputs from touch, key, accelerometer and compass to a RemoteInput at the given ip/port. 
Stage Container containing hierarchies of actors. 
UIWindow A 2D scene graph containing hierarchies of actors. 

Class Overview

An InputProcessor is used to receive input events from the keyboard and the touch screen (mouse on the desktop). For this it has to be registered with the setInputProcessor(InputProcessor) method. It will be called each frame before the call to render(). Each method returns a boolean in case you want to use this with the InputMultiplexer to chain input processors.
