Class Overview

2D scene graph node. An component has a position, rectangular size, origin, scale, rotation, Z index, and color. The position corresponds to the unrotated, unscaled bottom left corner of the component. The position is relative to the component's parent. The origin is relative to the position and is used for scale and rotation.

An component has a list of in progress actions that are applied to the component (often over time). These are generally used to change the presentation of the component (moving it, resizing it, etc). See act(float), Action and its many subclasses.

An component has two kinds of listeners associated with it: "capture" and regular. The listeners are notified of events the component or its children receive. The regular listeners are designed to allow an component to respond to events that have been delivered. The capture listeners are designed to allow a parent or container component to handle events before child components. See fire(Event) for more details.

An InputListener can receive all the basic input events. More complex listeners (like ClickListener and GestureListener) can listen for and combine primitive events and recognize complex interactions like multi-touch or pinch.

public boolean fire (Event event)

Sets this component as the event target and propagates the event to this component and ancestor components as necessary. If this component is not in the window, the window must be set before calling this method.

Events are fired in 2 phases.

  1. The first phase (the "capture" phase) notifies listeners on each component starting at the root and propagating downward to (and including) this component.
  2. The second phase notifies listeners on each component starting at this component and, if getBubbles() is true, propagating upward to the root.
If the event is stopped at any time, it will not propagate to the next component.

Returns
  • true if the event was cancelled.

public UIComponent hit (float x, float y, boolean touchable)

Returns the deepest component that contains the specified point and is touchable and visible, or null if no component was hit. The point is specified in the component's local coordinate system (0,0 is the bottom left of the component and width,height is the upper right).

This method is used to delegate touchDown, mouse, and enter/exit events. If this method returns null, those events will not occur on this UIComponent.

The default implementation returns this component if the point is within this component's bounds.

Parameters
touchable If true, the hit detection will respect the touchability.
