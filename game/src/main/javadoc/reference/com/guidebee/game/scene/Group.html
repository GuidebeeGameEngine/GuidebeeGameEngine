Class Overview

Container may contain other actors.

Actors have a z-order equal to the order they were inserted into the UIContainer. Actors inserted later will be drawn on top of actors added earlier. Touch events that hit more than one actor are distributed to topmost actors first.

public Actor hit (float x, float y)

Returns the deepest actor that contains the specified point and is touchable and visible, or null if no actor was hit. The point is specified in the actor's local coordinate system (0,0 is the bottom left of the actor and width,height is the upper right).

This method is used to delegate touchDown, mouse, and enter/exit events. If this method returns null, those events will not occur on this Actor.

The default implementation returns this actor if the point is within this actor's bounds.

public void setTransform (boolean transform)

When true (the default), the Batch is transformed so children are drawn in their parent's coordinate system. This has a performance impact because flush() must be done before and after the transform. If the actors in a UIContainer are not rotated or scaled, then the transform for the UIContainer can be set to false. In this case, each child's position will be offset by the UIContainer's position for drawing, causing the children to appear in the correct location even though the Batch has not been transformed.
